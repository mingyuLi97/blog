## TDD 优缺点

TDD（测试驱动开发） 是一种软件开发方式，开始开发前先编写测试用例。

### 优点

1. 角度转换：更多的站在用户的角度来看问题
2. 代码更稳定性、更优质
3. 易于维护，每次更改只需要查看测试用例是否完成
4. 易于重构

### 缺点

1. 测试的代码过多产生冗余，很可能测试代码比真实代码更多，而且会花费很多时间在想怎么测试
2. 前期投入时间过长，不适合紧急的软件开发
